# 每项目常驻非会话 Tab：Git Tab + Files Tab；默认激活为「运行中优先，否则 Tab 序」

ADR-0003 的 Tab 模型是「一个 Tab = 一个活的会话」，但 **Git Tab** 与 **Files Tab** 都不是会话——无进程、无输出流，不应被「关闭即弃会话」管辖。我们决定每个项目的 Tab 栏**常驻两个非会话 Tab**：**Git Tab** 恒排最前、**Files Tab** 紧随其后，均不可关闭（无 ×，Cmd+W 无效），与会话 Tab 共用激活/循环规则。未选常驻以外的按需打开方案，以免入口多一跳、关闭后状态语义分裂。

**默认激活**（首次进入项目 / 解析默认）：若有运行中的 **Run Session**，取 Tab 栏从左到右第一个运行中的；否则按 Tab 栏顺序（常驻下即 **Git Tab**）。冷启动不再优先已退出的会话 Tab——闲置时以 Git 为家，正在跑的会话仍一眼够得着。**关闭**激活 Tab 仍回落左邻、其次右邻，不套用默认激活规则。

## Considered Options

- **常驻非会话 Tab（选中）**：Git / Files 入口零操作；代价是 Tab 不再全是会话，「无 Tab 占位态」不存在。
- **按需打开 Files / 仅左树进文件**：不扩大常驻破例，但与 Git 工作流（打开文件）割裂，且关闭后是否保留编辑现场又生枝节。
- **默认激活：凡有会话 Tab 就进会话（旧）**：运行器气味浓，但会把人送进已退出的旧输出；改为运行中优先 + 否则 Tab 序。
- **关闭也按默认激活重算**：关终端可能跳到别的运行中会话，手感跳；故关闭仍邻接。

## Consequences

- `resolveTabs` 恒含 `gitKey` 与 Files Tab 键；顺序为 Git → Files → 运行会话（树序）→ 终端。
- 默认激活与关闭回落分离，见上文；每项目激活 Tab 的记忆可跨重启持久（ADR-0008），合法记忆优先于默认激活。
- `closeTab` / Cmd+W 对常驻非会话 Tab 为 no-op。
- Git / Files 的面板态均「切走隐藏不卸载」；Files 另持久化每项目上次打开路径与树展开状态（见 Files 相关 PRD）。
