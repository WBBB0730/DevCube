# 探测脚本来源进入引用键

Discovered Script 从「仅 `package.json` scripts」扩展为清单脚本与约定命令两类，同名可在不同来源下并存。引用型 Run Configuration 与会话唯一键因此从 `(Project, 名)` 改为 `(Project, 来源, 名)`；运行时按来源解析命令，不再假定一律 `pm run`。不做旧档案兼容——引用型必须带来源字段。

## Considered Options

- **键带来源、不兼容旧档案**（选中）：形状干净；用户极少，不必背读时缺省 / 迁移分支。
- **展示名加前缀挤进旧键**（如 `go:test`）：少改持久化形状，但污染展示名、与菜单分组语义不一致。
- **读时缺省 / 启动整库迁移**：曾考虑过为零感升级，最终因用户面小且易堆兼容代码而放弃。
