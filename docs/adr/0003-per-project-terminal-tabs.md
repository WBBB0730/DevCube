# 终端 Tab 以项目为维度，Terminal 独立于 Run Session

在运行控制台之外，允许用户在每个项目内自建任意多个终端。终端 Tab 栏**以项目为维度**：每个项目一套 Tab——Tab 1 是跟随该项目选中配置的运行控制台，其后是该项目的自由终端；而非全局共享一条终端栏。**Terminal**（项目根目录下的交互 shell）不绑定任何 Run Configuration，与 Run Session 并列但独立；壳可按项目持久化，进程与输出不持久（见 ADR-0008）。

## Considered Options

- **每项目一套 Tab（选中）**：终端归属项目、随「当前项目」切换，贴合本工具「以项目为维度」的定位；一个项目的运行控制台与它的终端聚在一起。代价：引入「当前项目」状态，左树项目行由「整行折叠」改为「单击选中、点箭头折叠」。
- **全局一套终端栏**：改动小、无需「当前项目」概念，但项目维度只落在各终端的 cwd 上，终端不随项目聚合、切项目时仍混在一条栏里，与整体模型不一致。

## Consequences

- 新增「当前项目」状态；项目行成为可选中的一等对象（高亮当前项目），折叠交互迁移到左侧箭头。
- **Terminal** 的壳（id / 名字 / 顺序）可按项目持久化（见 ADR-0008）；**进程与输出**仍不持久——重启后空 shell，第一次激活该 Tab 才 spawn。shell 自行结束即自动关闭其 Tab，并与落盘集合同步删除。与 ADR-0002「Run Session 不持久化」仍一致（运行 Tab 不靠落盘复活）。
- Terminal 不参与 Run Session 的单实例去重——每个终端各自独立，同一项目可并存任意多个；移除项目时一并杀掉并清除其全部终端（含落盘壳）。

## 修订（2026-07）

运行控制台不再是单一的「Tab 1 跟随选中配置」——改为**每条有会话的配置各占一个 Tab**（一个 Tab = 一个活的会话视图，与 Terminal 并列；运行会话组按树序在前、终端组在后）。「每项目一套 Tab 栏」的主决策不变。原因：单一控制台「跟随选中原地换内容」与 Tab 隐喻互相打架，激活规则反复调整仍难自洽；且无法并列查看并发运行的多条配置输出。已退出的会话保留 Tab（可回看输出）直到手动关闭，关闭即弃会话。

## 修订（2026-07 · 工作台持久化）

原「Terminal 纯内存、重启即消失」改为：壳落盘、进程懒起；详见 ADR-0008。每项目一套 Tab 栏的主决策不变。
