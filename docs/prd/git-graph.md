## Problem Statement

我在 Runlet 里聚合了多个项目来运行命令、看输出，但想了解某个项目的 git 状态（历史走向、分支、某次提交改了什么、要不要合并/回滚）时，必须切去 IDE 或终端。项目维度的工作台缺一块「仓库视图」，上下文来回切换很割裂。

## Solution

每个项目的 Tab 栏常驻一个 **Git Tab**（排最前、不可关闭）：完整移植 vscode-git-graph 的能力——提交图谱（分支线 + 圆点 + 引用标签）、提交/贮藏/未提交更改的详情与文件变更、应用内 diff 面板、右键执行全套 git 操作（分支/标签/合并/变基/重置/拣选/贮藏/远程同步）、分支筛选、提交查找、仓库设置。数据由主进程直接跑 git CLI 产出；`.git` 变动自动刷新图谱。

## User Stories

1. 作为开发者，我想在项目的 Tab 栏里直接打开 Git Tab，这样不用离开 Runlet 就能看仓库历史。
2. 作为开发者，我想看到提交历史的图形化分支线与每行的分支/标签/贮藏标签，这样一眼读懂仓库拓扑。
3. 作为开发者，我想看到「未提交的更改 (N)」行，这样知道工作区是否干净。
4. 作为开发者，我想点一个提交展开详情（作者/提交者/日期/完整信息/改动文件树），这样了解这次提交做了什么。
5. 作为开发者，我想点详情里的文件直接看 diff（应用内渲染），这样不依赖外部编辑器。
6. 作为开发者，我想 Ctrl/Cmd+点第二个提交进入比较模式，这样看两点之间的全部文件差异。
7. 作为开发者，我想右键提交创建分支/打标签/拣选/回滚/丢弃/重置，这样常用操作不用敲命令。
8. 作为开发者，我想右键分支标签执行检出/重命名/删除/合并/变基/推送，这样分支管理都在图上完成。
9. 作为开发者，我想双击本地分支标签直接检出、双击远程分支标签创建跟踪分支，这样高频操作最短路径。
10. 作为开发者，我想右键远程分支执行拉取/获取到本地分支/删除远程分支。
11. 作为开发者，我想对贮藏执行应用/弹出/丢弃/建分支，并在图上看到贮藏节点。
12. 作为开发者，我想右键未提交行进行贮藏/重置/清理未跟踪文件。
13. 作为开发者，我想用工具栏的分支下拉筛选图谱显示的分支（多选、显示全部）。
14. 作为开发者，我想勾选「显示远程分支」开关控制远程引用的显示。
15. 作为开发者，我想点「刷新」从所有远程 fetch 并重新加载图谱（本地先出、远程随后）。
16. 作为开发者，我想 Cmd/Ctrl+F 搜索提交（消息/hash/作者/引用，支持大小写与正则），命中行高亮并可跳转。
17. 作为开发者，我想在仓库设置里管理 remote（增删改/清理）、编辑 user.name/email、隐藏某些 remote。
18. 作为开发者，我想让危险操作（删除分支/重置/丢弃提交等）先弹确认对话框，避免误操作。
19. 作为开发者，我想在分支未合并导致删除失败时得到「强制删除」的追问，而不是原始报错。
20. 作为开发者，我想在推送标签时得到「提交不在远程」的预警。
21. 作为开发者，我想操作失败时看到 git 的原始错误信息。
22. 作为开发者，我想 `.git` 目录变化（外部提交/切分支）后图谱自动刷新，且刷新不打断我展开的详情。
23. 作为开发者，我想图谱「加载更多」或滚动到底自动加载更早的提交。
24. 作为开发者，我想对非 git 仓库的项目看到明确的兜底提示而不是报错。
25. 作为开发者，我想交互式变基在项目的 Terminal Tab 里启动，这样可以在终端里完成编辑。
26. 作为开发者，我想注释标签能查看 tagger 与消息详情。
27. 作为开发者，我想从提交详情里打开工作区文件、复制文件路径。
28. 作为开发者，我想把某文件重置到某个提交时的状态。
29. 作为开发者，我想在工具栏左侧的「视图选项」里切换图谱显示（远程分支/贮藏/标签/reflog/首父/提交排序），这样不必进仓库设置弹窗。
30. 作为开发者，我想在分支下拉里选「全部分支」或「当前分支」，「当前分支」跟随 HEAD（checkout 后自动跟随），这样快速聚焦当前工作分支。
31. 作为开发者，我想从工具栏直接刷新（fetch + 静默重载，本地先出、远程随后）、拉取、推送（可强制）、创建分支，这样常用远程/分支操作不必进右键菜单。
32. 作为开发者，我想选中「未提交更改」行后看到一个提交面板（左：提交信息 + 按钮，右：已暂存/未暂存文件），这样不离开 Runlet 就能提交。
33. 作为开发者，我想勾选文件把它加入已暂存区、取消勾选移回未暂存区（对齐 SourceTree），这样精确控制本次提交的内容。
34. 作为开发者，我想暂存某文件后继续修改它时，已暂存区仍是暂存那一刻的快照、未暂存区显示后续改动，这样部分暂存语义正确。
35. 作为开发者，我想点已暂存文件看 HEAD→暂存 的差异、点未暂存文件看 暂存→工作区 的差异。
36. 作为开发者，我想写提交信息后「提交」，或勾「修正」改上次提交，或用「提交并推送」一步同步（推送时可选强制），这样提交与推送连贯完成。
37. 作为开发者，我想从文件行的 `…` 菜单撤销更改 / 删除未跟踪文件 / 打开文件 / 复制路径。
38. 作为开发者，我想暂存 / 取消暂存 / 撤销都静默即时、不打断我看图，这样操作顺滑。

## Implementation Decisions

- **入口与 Tab 模型**：Git Tab 是每项目常驻、非会话的 Tab，键 `git:<projectPath>`，恒排最前、不可关闭；与会话 Tab 共用激活/循环/回落规则（ADR-0005）。面板「切走隐藏不卸载」，首次可见才拉数据。
- **数据源**：主进程直接 spawn git CLI（不经 shell），git 路径经用户登录 shell 解析一次并缓存（GUI 应用 PATH 缺失）；所有解析基于机器可读输出（自定义 --format 分隔符、-z NUL 分隔）；解析器是纯函数、与 IO 编排分层。
- **对齐策略**：UI 层为 React 重写（原项目是命令式 DOM + webview，不可复用）；**纯逻辑层以参考项目原源码为唯一权威**——git 命令构造、解析规则、布局算法、菜单可见性、对话框默认值逐函数对照原实现，能照抄的函数体照抄（仅剥 vscode 依赖、调整类型），行为偏差按缺陷修正。
- **IPC 契约**：读操作独立通道（load/details/file-diff/tag-details/repo-config/settings）；约 30 个写操作走单通道判别联合 GitAction，结果为判别联合（ok / error / push-tag-not-on-remote）；仓库变化推 git:changed 事件，渲染端只对已加载过的项目软刷新。
- **加载语义**：一次 load 主进程内部串联仓库概要 + 提交列表 + refs 返回；初始 300 条、加载更多 +100 后整表重拉（哨兵 N+1 判断还有更多）；软刷新保持画面原地换数据，硬刷新（筛选/设置变更/重试）清空重来；空仓库与非仓库是明确的状态而非错误。
- **图谱布局**：忠实移植参考项目的 Vertex/Branch/determinePath 算法为纯函数（网格坐标 → SVG 路径），12 色调色板循环，颜色经 CSS 变量引用。
- **详情与 diff**：详情面板 v1 采用吊底停靠（非参考默认的行内嵌入——React 表格行内嵌套复杂度高，作为已知取舍）；diff 为应用内 unified 渲染，数据是主进程解析好的结构化 hunks（git diff -U3，二进制/超大文件兜底）；比较方向恒按行序归一化（老→新），未提交端恒为 to。
- **写操作安全**：动作对话框与追问链对齐参考（强删确认、重名替换、push tag 预检）；「不再提示」标记持久化于跨项目视图偏好；动作执行期间文件监听静音（含 1500ms 余震），完成后主动推刷新。
- **凭证**：依赖系统凭证助手（ssh-agent/钥匙串），注入 GIT_TERMINAL_PROMPT=0 快速失败，错误原样呈现；不做 askpass。
- **交互式变基**：不在主进程跑——渲染端在该项目新建 Terminal 并写入 `git rebase --interactive` 命令行。
- **持久化**：每项目 git 设置（显示开关三态/排序/隐藏 remote）与跨项目视图偏好统一存 electron-store（ADR-0002 同一份 JSON）。
- **不做头像**（无网络依赖）；**不做 GPG 签名校验**（tag 详情仅展示是否签名）。
- **工具栏重排**：右侧操作钮 = 刷新 / 拉取 / 推送 / 创建分支 / 搜索 / 设置。「刷新」= `git fetch --all` + 静默软重载（本地图谱先原地更新、远程 fetch 完成后再软刷新一次；全程不置全局 loading、不弹进行中遮罩），取代原独立「获取」钮。拉取 / 推送 / 创建分支复用既有对话框，预设为当前分支 / HEAD；detached 或无上游时拉取/推送置灰；推送对话框已含 常规 / force-with-lease / 强制。
- **视图选项 Popover**：图谱显示三态开关（远程分支/贮藏/标签/reflog/首父）+ 提交排序从「仓库设置」移出，收进工具栏**左侧、紧挨分支下拉**的 Popover（工具栏原重复的「显示远程分支」checkbox 并入）。分支下拉浮层项「显示全部」改文案「全部分支」，并加「当前分支」= filter 传 `HEAD` ref 动态跟随（不持久化，随重启回落全部）。
- **仓库设置瘦身**：仅保留 git 级配置（用户信息、远程管理、隐藏远程）。删除死配置「显示名」（从不被读）与「Issue 链接」整功能（含 `tokenizeBody` 的 issue 层、`IssueLinkingConfig` 类型/字段/校验）；提交信息中的 URL / 邮箱自动链接保留。
- **日期中文化**：`formatDateTime` 改用 `Intl.DateTimeFormat('zh-CN')` 输出「YYYY年M月D日 HH:MM:SS」（零依赖标准 API，去掉手写月份表）；提交表格「日期」列仍为中文相对时间。
- **未提交更改的暂存与提交**（见 ADR-0006）：未提交行详情从只读升级为带暂存区的提交面板（版式 A：左提交信息框 + 「提交」/下拉「提交并推送」+「修正」勾选，右已暂存/未暂存两段文件树）。数据模型改为 `git diff --cached`（已暂存 HEAD↔index）+ `git diff`（未暂存 index↔工作区）+ 未跟踪，同一文件可同时在两段；diff 面板支持 index 端点。勾选 = `git add`、取消勾选 = unstage、区头 = 全部；写操作 stage/unstage/discard/commit(amend) 静默即时，提交提交 index，「提交并推送」提交后打开 push-branch 对话框（含强制）。仅文件级暂存。文件行 `…` 菜单：未暂存 = 撤销更改…（未跟踪则删除文件…）/ 打开文件 / 复制路径；已暂存 = 取消暂存 / 打开文件 / 复制路径。图上未提交行的行级右键菜单（贮藏/重置/清理）不变。

## Testing Decisions

只测纯函数、零 mock（项目既有约定）：git stdout 解析器用真实样本字符串（含二进制 numstat、rename 跨 NUL 段、annotated tag 双行、空仓库等参考实现的已知坑）；动作层测参数构造纯函数与错误聚合语义；布局引擎测列分配/线段坐标/路径 d 串到具体数值；日期格式化测阈值边界。先例：src/main/*.test.ts 与 src/shared/runnable.test.ts 的「构造输入 → 断言输出」风格。不写 UI/e2e 测试。新增：暂存/未暂存/未跟踪的 git 输出解析（`git diff --cached` / `git diff` / status）用真实样本测（含同一文件同时落在两段的场景），沿用纯函数零 mock 约定；stage/unstage/commit 动作层测参数构造；日期格式化改测中文 Intl 输出的边界串。

## Out of Scope

- Pull Request 创建（提供方模板配置整块不做，设置形状留 null 占位）
- Code Review 跟踪（已读文件标记）
- 作者头像、GPG 签名校验、askpass 凭证交互
- 提交列表列宽拖拽与列显隐持久化、refs 标签三种对齐模式（v1 固定 Normal）
- 消息内 emoji shortcode / markdown 富渲染（v1 纯文本 + URL/邮箱自动链接）
- 行内嵌入式详情（v1 吊底停靠）
- 多仓库/子模块发现（每项目即其根仓库）
- Issue 链接（提交信息 issue 号链接化）—— 整功能移除（URL / 邮箱自动链接保留）
- 部分暂存（hunk / 行级 staging）—— 仅文件级
- 文件夹级复选框 —— 暂存靠单文件勾选或区头「全部」
- 分支筛选持久化（含「当前分支」模式）—— 纯内存，随重启回落「全部分支」

## Further Notes

- 术语见 CONTEXT.md（Git Tab）；Tab 模型破例的取舍见 ADR-0005。
- 参考实现为 vscode-git-graph，其许可为**自定义受限许可（非 OSI 开源）**。本实现是行为对齐的重写（TypeScript/React 技术栈不同、无源码拷贝），但图谱布局算法为语义忠实移植；Runlet 若对外分发需自行评估该许可风险（个人本地使用风险低）。
- 深色配色中 diff 与图谱色为 Darcula 系推导值，DESIGN.md 標注待校准。
- 未提交的暂存与提交面板行为对齐 SourceTree（暂存快照语义、已暂存/未暂存两段管理）；术语见 CONTEXT.md（未提交更改行），取舍见 ADR-0006。DESIGN.md 的 Git 图谱节（工具栏 / 详情面板 / 图标）将在实现 UI 时同步更新。
