## Problem Statement

我在 DevCube 里跑项目、看 Git，改一行配置或源码却还要切到 WebStorm / 系统编辑器。Git 变更树里的「打开文件」也只丢给系统默认应用，和面板内的工作流是断的。我需要在不离开 DevCube 的前提下，浏览项目里的文件并直接编辑。

## Solution

每个 **Project** 的 Tab 栏增加常驻、不可关闭的 **Files Tab**（标签：图标 +「文件」），排在 **Git Tab** 之后、会话 Tab 之前：

1. **左正文 + 右文件树**：树展示项目根下条目；**不按 `.gitignore` 过滤**（仍可见 `node_modules` 等），但隐藏 WebStorm「Ignored Files and Folders」默认项（如 `.git`、`.DS_Store`）；同一时刻至多打开一个条目。
2. **按类型分流**：文本用编辑器编辑（当前实验分支为 CodeMirror 6；`backup/files-tab-monaco` 为 Monaco 对照——均仅语法高亮与文本编辑，无 lint / 补全 / 跳转；不上外部 LSP）；图片内嵌预览；其余只读占位，并可「在其他应用中打开」。
3. **事件自动保存**（对齐 WebStorm 体感）：换文件 / 离开 Files Tab / 失焦或短空闲时写入；无显式脏标记弹窗常态。
4. **与 Git 打通**：Git 里「打开文件」进入该项目 Files Tab 并打开对应路径；Files Tab 保留「在其他应用中打开」。
5. **左栏不动**：ProjectTree 仍只做项目 / 配置；文件树仅出现在 Files Tab 内。不做文件 CRUD（新建 / 重命名 / 删除 / 复制 / 移动仍走系统文件管理器或 **Terminal**）。

## User Stories

1. 作为开发者，我想在项目 Tab 栏看到常驻的 **Files Tab**（「文件」），以便不用找入口就能浏览项目文件。
2. 作为开发者，我不想关掉 Files Tab，以便它和 **Git Tab** 一样始终在。
3. 作为开发者，我想 Files Tab 排在 Git 之后、运行会话与终端之前，以便两个常驻工具面挨在一起。
4. 作为开发者，我想用 `Ctrl+Tab` / `Ctrl+Shift+Tab` 也能循环到 Files Tab，以便键盘在全部 Tab 间移动。
5. 作为开发者，我想在 Files Tab 上按 `Cmd/Ctrl+W` 无效，以便不会误关常驻 Tab。
6. 作为开发者，我想看到项目根下接近完整的文件树（含 `node_modules` 等 gitignore 条目），以便不被 `.gitignore` 藏掉我要找的文件；同时不想看到 `.git` / `.DS_Store` 等 IDE 默认忽略项，以便和 WebStorm Project 树一致。
7. 作为开发者，我想展开 / 折叠目录浏览树，以便逐级找到文件。
8. 作为开发者，我想点击文件就在右侧打开它，以便立刻查看或编辑。
9. 作为开发者，我想同一时刻只打开一个条目，以便界面保持轻量、没有内层文件 Tab。
10. 作为开发者，我想点另一个文件时自动处理当前缓冲（事件保存），以便换文件不丢已键入内容、也不弹窗打断。
11. 作为开发者，我想用 Monaco 编辑文本文件，以便有语法高亮和基本编辑体验。
12. 作为开发者，我想打开文本文件时只有语法高亮与基本编辑，不要补全、悬停、跳转或诊断，以免单文件无工程上下文时误导。
13. 作为开发者，我不想接外部 LSP / 语言服务器，以便 Files Tab 不膨胀成迷你 IDE。
14. 作为开发者，我想打开图片时在右侧内嵌预览，以便不用跳出去看资源。
15. 作为开发者，我想打开无法在此编辑的二进制时看到占位说明，以便知道为什么不能改。
16. 作为开发者，我想在 Files Tab 里一键「在其他应用中打开」当前条目，以便预览不够时交给系统应用。
17. 作为开发者，我想点工具栏面包屑中的目录或文件名时，在右侧文件树展开并滚到对应行，以便从路径快速定位树节点。
18. 作为开发者，我想一键「在文件树中显示」当前打开的文件，以便树滚偏后仍能回到该文件行。
19. 作为开发者，我想从「最近打开文件」菜单点回最近打开过的最多 10 个文件，以便快速回到刚改过的文件（空列表时提示暂无）。
20. 作为开发者，我想从 Git 变更树 / 提交面板「打开文件」时进入 Files Tab 并打开该路径，以便 Git 与编辑不断档。
21. 作为开发者，我想未打开任何条目时提示「在右侧选择文件」，以便知道下一步点哪里。
22. 作为开发者，我想换文件或离开 Files Tab 或失焦/短空闲时自动保存文本，以便接近 WebStorm 的保存体感。
23. 作为开发者，我想磁盘上文件被外部改掉且我没有未落盘缓冲时静默重载，以便跟得上 Terminal / 其他编辑器的改动。
24. 作为开发者，我想在「还有未落盘改动且磁盘也变了」时被提示选择重载或保留编辑器内容，以便不静默丢键入。
25. 作为开发者，我不想在 Files Tab 里新建 / 重命名 / 删除 / 复制 / 移动文件，以便管理操作仍走 Finder 或 Terminal、保持轻量。
26. 作为开发者，我想左栏仍然只显示项目和运行配置，以便运行器主路径不被文件树占据。
27. 作为开发者，我想重启后恢复该项目上次打开的路径、树展开状态与最近打开列表，以便接着改同一个文件。
28. 作为开发者，我想上次路径已不存在时静默回到空态（树展开仍尽量恢复；最近打开里失效项剔除），以便不弹恼人错误。
29. 作为开发者，我想首次进入项目时：若有运行中的 Run Session 则激活第一个运行中的，否则按 Tab 序落到 Git Tab，以便闲置看 Git、有活进程看输出。
30. 作为开发者，我想关闭某个会话 Tab 时仍落到左邻（其次右邻），以便关 Tab 手感不跳去别的运行中会话。
31. 作为开发者，我想切走 Files Tab 再切回来时编辑现场还在（未触发保存前的缓冲与滚动位置按「隐藏不卸载」保留），以便和 Git / xterm 一致。
32. 作为开发者，我想保存失败时得到明确反馈，以便知道磁盘权限或路径问题。
33. 作为开发者，我不想一期做快速打开（Cmd+P）或树顶过滤，以便先交付核心浏览+编辑（大项目靠树展开点选）。

## Implementation Decisions

- **Tab 模型**：Files Tab 为每项目常驻非会话 Tab，键形态与 Git 并列（如 `files:<projectPath>`），恒排第二、不可关闭；`closeTab` / Cmd+W no-op。顺序与默认激活 / 关闭回落见 ADR-0005（已替换为「Git + Files 常驻」+「运行中优先否则 Tab 序」）。
- **布局**：Files Tab 内左正文、右文件树；左栏 ProjectTree 不改。空态文案「在右侧选择文件」。标签：`FolderOpen` 图标 +「文件」，内边距对齐 Git Tab。工具栏：可点面包屑（点段 → 树展开并滚到对应行）+「最近打开文件」（最多 10 条，按项目持久化）/「在文件树中显示」/「在文件夹中显示」/「在其他应用中打开」。文件树顶栏：「全部展开」/「全部折叠」。
- **文件树**：列出项目根下条目；**不读 `.gitignore`**。隐藏名对齐 WebStorm「Editor → File Types → Ignored Files and Folders」默认掩码（`.git`、`.DS_Store`、`*.pyc`、`*~` 等；完整表见共享过滤模块）。仅允许访问项目根之内的路径（防目录穿越）。目录懒加载或等价按需读取以控制大树成本；展开状态按项目持久化。
- **打开分流**：按类型（扩展名 / MIME / 内容嗅探，实现时择稳妥策略）分为文本 → Monaco；图片（如 png/jpg/gif/webp，svg 若按文本更合适可走 Monaco）→ 内嵌预览；其余 → 占位 +「在其他应用中打开」（复用既有系统打开能力）。
- **编辑器**：实验分支用 CodeMirror 6（`@uiw/react-codemirror` + 语言包高亮，`basicSetup.autocompletion/lint` 关闭）。Monaco 完整实现保留在分支 `backup/files-tab-monaco` 供对照。明确不引入外部 LSP。查找等纯编辑器能力可用。
- **保存**：事件自动保存——至少覆盖：切换打开条目、离开 Files Tab、窗口/面板失焦、短空闲。无常态「保存/不保存/取消」三按钮；竞态（未落盘 + 磁盘变更）单独弹窗：重载 / 保留编辑器内容。
- **外部变更**：无未落盘缓冲时静默以磁盘为准刷新当前视图；有未落盘且磁盘变了走竞态弹窗。
- **持久化**：每项目记住上次打开路径 + 树展开状态 + 最近打开（最多 10，新→旧），走既有集中配置存储（ADR-0002）。路径无效 → 静默空态 / 从最近列表剔除，不弹错。打开缓冲本身不要求跨重启持久（靠自动保存落盘）。
- **Git 集成**：Git 文件行「打开文件」改为聚焦该项目 Files Tab 并打开路径；系统打开改由 Files Tab（及二进制占位）上的「在其他应用中打开」承担。提交面板 / 详情菜单文案与行为对齐此分流。
- **IPC**：主进程提供项目范围内的目录列举、读文件、写文件、以及已有的系统打开；读写真值限定在项目根下。渲染层 Files 状态（当前路径、展开集、面板挂载）与 Git store 类似按项目分桶，切走隐藏不卸载。
- **默认激活**：实现时替换旧「首个运行会话 → 首个终端 → Git」缺省逻辑为 ADR-0005 新规则；关闭邻接逻辑保持。

## Testing Decisions

好测试：只测外部可观察的纯函数 / 契约行为，不测 Monaco 挂载与像素布局。

要测的模块（建议）：

- **路径安全**：解析/归一化后必须落在项目根内；拒绝 `..` 穿越与根外绝对路径。
- **树忽略名**：给定条目名 → 是否按 IDE 默认 Ignored Files 隐藏（含正例 `.git` / `.DS_Store`、反例 `node_modules` / `.env`）。
- **打开分流判定**：给定扩展名/样本 → 文本 / 图片 / 其它（纯函数表驱动）。
- **默认激活解析**：给定 Tab 列表 + 各会话是否 running → 期望激活键；关闭邻接用例与默认激活分离。
- **持久化形状**：缺省字段、无效路径清空当前打开、展开集读写（与现有 store 补齐路径一致的风格）。

不写 Monaco/e2e UI 测试。先例：`runnable` / `project-sort` / git 解析类 vitest「构造输入 → 断言输出」。

## Out of Scope

- 快速打开（Cmd+P）/ 项目内文件名过滤 / 全文搜索
- 内层多文件 Tab / 分屏编辑
- 文件 CRUD（新建、重命名、删除、复制、移动、拖拽）
- 外部 LSP / 语言服务器 / 跳转定义跨文件工程级智能
- 左栏改为文件树或配置/文件双模切换
- 应用内三方合并编辑器（仍见 git-graph Out of Scope）
- 按 `.gitignore` 默认隐藏（明确不做；与「IDE Ignored Files」默认掩码是两回事）
- 协作编辑、断线同步、文件锁定

## Further Notes

- 术语见 CONTEXT.md（**Files Tab**）；Tab 破例与默认激活见 ADR-0005；视觉见 DESIGN.md。
- 「轻量」在此指少写业务胶水、不做文件管理与语言智能——不指捆绑包体积。编辑器壳在 Monaco / CM6 间实验对照后再定稿。
- 保存触发的具体空闲秒数与失焦边可在实现时按 WebStorm 默认体感微调，不必再开产品讨论，除非要做成用户设置。
