## Problem Statement

左树项目一多，常用 **Project** 容易被埋在下面。现有排序（自定义 / 名称 / 添加时间 / 打开时间）和拖拽只能临时改序，没法把「我天天用的那几个」在任意排序下都钉在上面，滚动长列表时项目名也会滚出视口，还得翻回去找。

## Solution

给每个 **Project** 增加持久的 **Pin（置顶）** 标记：

1. **排序浮顶**：已置顶的项目整段（项目行 + 其下配置）排在未置顶之前；两组内部仍服从当前排序方式。
2. **滚动吸顶**：滚动左树时，所有已置顶的**项目行**在列表顶依次叠放保持可见（行间 1px）；未置顶的**项目行**滚过时贴在置顶堆下方作为当前段、被下一段顶走；配置行照常滚走。
3. **显式开关**：仅通过项目「更多」/ 右键菜单置顶或取消置顶；已置顶在名称旁常显小图钉。

## User Stories

1. 作为用户，我想给常用 **Project** 打上 **Pin**，以便它在任意排序下都整段排在未置顶项目之上。
2. 作为用户，我想取消某个项目的 **Pin**，以便它回到未置顶区块并按当前排序排列。
3. 作为用户，我想在项目行的「更多」菜单里看到「置顶」或「取消置顶」，以便与打开文件夹 / 新建终端等操作同一入口。
4. 作为用户，我想用项目行右键打开与「更多」相同的菜单并开关 **Pin**，以便不必找 ⋮ 按钮。
5. 作为用户，我想已置顶项目在名称旁常显小图钉图标，以便一眼认出谁被 Pin 了。
6. 作为用户，我想 **Pin** 跨重启保留，以便不用每次重开再钉。
7. 作为用户，我想新添加的项目默认不 Pin，以便置顶区只保留我明确常用的。
8. 作为用户，我想置顶时该项目进入置顶区块的开头，以便新钉的立刻出现在置顶区最前。
9. 作为用户，我想取消置顶时该项目进入未置顶区块的开头，以便行为与置顶对称、可预期。
10. 作为用户，我想在名称 / 添加时间 / 打开时间排序下，置顶组与未置顶组内仍各自按该规则排，以便 Pin 不破坏组内排序语义。
11. 作为用户，我想在自定义排序下拖拽重排时只能在置顶组内或未置顶组内移动，以便拖拽不会误改 Pin。
12. 作为用户，我想拖过置顶/未置顶边界时无效（松手弹回或不落盘），以便边界密封。
13. 作为用户，我想非自定义模式下组内拖拽仍在松手且顺序实质变化后切到自定义，以便与现有拖拽改序习惯一致（且仍不跨界）。
14. 作为用户，我想筛选项目名时，匹配结果里仍是「匹配的置顶在上、匹配的未置顶在下」，以便筛选不吞掉 Pin 分区。
15. 作为用户，我想筛选开启时拖拽仍禁用，但菜单里仍可开关 Pin，以便筛选只拦拖拽不拦置顶。
16. 作为用户，我想往下滚左树时所有已置顶的项目行叠在列表顶部保持可见，以便不用滚回顶部找置顶项目。
17. 作为用户，我想置顶叠放行之间留 1px 间隙，以便叠层仍可分辨边界。
18. 作为用户，我想吸顶只作用于项目行、配置行照常滚走，以便展开的配置不会粘在顶上占空间。
19. 作为用户，我想多个置顶时项目行在顶上依次叠放（叠罗汉），以便每个置顶项目名都仍可见。
20. 作为用户，我想在排序菜单里开关「固定置顶」，以便关掉多 Pin 叠放吸顶；关掉后视口最上的项目行仍作为当前段吸顶（默认开，跨重启保留）。
21. 作为用户，我想滚过项目时其项目行贴在视口顶（开启固定置顶时未置顶贴在置顶堆下方），被下一个项目顶走，以便视口顶被切一半的标题会吸住而不是半截悬挂。
22. 作为用户，我想点击已吸顶的置顶项目行时列表滚到该项目在文档中的位置，以便从叠放条回到它的配置区（sticky 在视口内时普通 scrollIntoView 无效）。
23. 作为用户，我想在打开时间排序下打开/选中某项目后列表滚到该项目（可见即可），以便有 Pin 时滚到列表顶不再保证看到刚打开的项。
24. 作为用户，我想添加项目成功后的滚入视口行为保持 `nearest`（置顶则按布局位置），以便与打开时间滚动策略一致。
25. 作为用户，我想移除项目时其 Pin 一并消失；同路径再添加视为新项目且默认不 Pin，以便不留幽灵状态。
26. 作为用户，我不希望行内再占一个图钉按钮槽，以便保持项目行克制（折叠 + 更多为主）。

## Implementation Decisions

- **数据模型**：`Project` 增加持久布尔字段 `pinned`（域名 **Pin**）。老档案缺省视为 `false`；读取层与既有 `addedAt` / `lastOpenedAt` 补齐同一路径，脏则回写。不另建置顶路径表。
- **排序**：共享纯函数（现有项目排序模块）先按 `pinned` 分成两区（置顶在前），再在各区内套用当前 `ProjectSortPrefs`（自定义 = 该区内落盘相对序；名称 / 添加时间 / 打开时间 = 既有比较规则）。筛选仍先排后滤，或等价地在过滤结果上保持同分区规则。
- **Pin / 取消 Pin 与落盘序**：任意当前排序模式下，开关 Pin 都写 `pinned`，并改写落盘 `projects[]`——置顶则移到「当前所有已置顶」之前（置顶区开头），取消则移到「当前所有未置顶」之前（未置顶区开头）。不因此自动切换排序 mode。
- **新项目登记**：继续默认不 Pin；插入策略与「未置顶区开头」一致（与现有插到数组头 + 分区排序相容）。
- **IPC**：渲染↔主增加置顶/取消置顶（或统一 `setProjectPinned(path, pinned)`）；返回更新后的项目树。持久化走既有集中配置存储。
- **拖拽**：自定义序（及拖拽触发切自定义）时，dnd 只允许在同一 `pinned` 分区内重排；跨区 drop 不改序、不改 Pin。筛选中仍禁用拖拽。
- **菜单**：项目「更多」/ 右键共用菜单增加「置顶」/「取消置顶」（图标 `Pin` / `PinOff`），放在「移除项目」之上；无行内图钉按钮。
- **视觉**：已置顶时项目名旁常显小 `Pin` 图标（muted）；未置顶无图标。包管理器角标与 hover 让位规则不变，图钉与角标并排时以不挤爆单行为准（实现时按 DESIGN 密度对齐）。
- **滚动吸顶**：左树滚动容器内，已置顶项目的**项目行**默认叠放吸顶（`top: n×(行高+1px)`，行间 1px 用不透明面板底填满）；可由排序菜单「固定置顶」关闭（`projectSortPrefs.pinSticky`，默认 `true`，老档案缺省当 true）。关闭后置顶行改为当前段吸顶（`top: 0`），与未置顶一致——视口最上的项目行贴住、被下一段顶走。开启时未置顶行贴在整叠置顶下方。仅项目行，不含配置行 /「检测到的配置」触发行。吸顶行需不透明底。点击已吸顶置顶标题时滚前方非 sticky 段锚（`scrollIntoView({ block: 'start' })`，`scroll-margin-top` 预留叠放高度）。
- **打开时间滚动**：打开时间模式下选中/打开项目后，改为将该项目滚入视口（未置顶 `nearest`；置顶同点击，走段锚 `scrollIntoView`），不再特供「列表 scrollTop = 0」。

## Testing Decisions

好测试：只测外部可观察的纯函数行为，不测吸顶 CSS / dnd 命中细节。

要测的模块（扩展既有项目排序单测）：

- **分区排序**：置顶整段在前；组内名称 / 添加时间 / 打开时间 / 自定义相对序正确；`pinned` 缺省当 false。
- **Pin 落盘位移**（可提纯）：置顶 → 置顶区开头；取消 → 未置顶区开头；多次开关后数组相对序稳定可测。
- **筛选 + 分区**：过滤后仍保持置顶在前。

不写 UI/e2e（吸顶叠放、跨界拖拽弹回靠手动验证）。

先例：`project-sort` 的 vitest 纯函数测试。

## Out of Scope

- 行内一键图钉按钮
- 置顶区与未置顶区之间的分隔线
- 置顶数量上限
- 添加项目流程里的「同时置顶」选项
- 拖拽跨界自动改 Pin
- 项目分组 / 多级收藏夹
- 键盘快捷键置顶
- 为 Pin 单独建 ADR（可逆、不令人意外）

## Further Notes

- 术语见 CONTEXT.md（**Pin**）；口语里「新项目碰巧排到最前」不是 Pin。
- 视觉与交互落点同步更新 DESIGN.md（菜单项、图钉、分区拖拽、吸顶、打开时间滚动）。
- 吸顶与现有「拖拽中全部暂时收起」同时存在时，以拖拽手感优先；松手后恢复吸顶即可，不另发明拖拽期特殊 Pin 语义。
